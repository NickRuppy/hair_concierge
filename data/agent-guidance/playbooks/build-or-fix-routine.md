When to use: routine structure. Use this when the user asks what the routine should look like, how to simplify it, or which steps belong where.

What to do: start from authoritative routine tool output, then make the answer explicit about what to keep, add, reduce, and treat as optional. Use short action-led labels for each step. Prefer simplification when crowded, and explain the routine in a way that makes the core path obvious before the extras. Ask at most one targeted follow-up if an important detail is missing.

What to avoid: do not invent extra routine steps, do not present optional steps as mandatory, do not bury the simplification advice, do not use this for a narrow technique question that only needs usage steps
